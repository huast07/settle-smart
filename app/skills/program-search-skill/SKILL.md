# Program Search Skill

Use this skill when the workflow has a destination, priority categories, and search queries, and needs candidate programs for later eligibility, verification, signup, and recommendation steps.

## Workflow

1. Read the destination and each priority category.
2. Search category by category. Do not let one category, such as grocery or banking, crowd out the rest.
3. For every essential goods or service category, try to find at least three candidate loyalty, rewards, member, or discount programs when current evidence exists.
4. Search official sources first: retailer, grocery chain, pharmacy, bank, credit issuer, transit agency, city/province/state, utility, telecom carrier, internet provider, clinic, and provider signup pages.
5. For groceries, first identify named chains that actually serve the destination, then search each chain's official loyalty, rewards, member pricing, or digital coupon page.
6. Keep non-loyalty essential programs only when they directly support newcomer settling needs and clearly label them as discount, public, utility, or member programs.
7. For cellular and telecommunications, search local mobile carriers, prepaid plans, internet providers, public affordability programs, multi-line offers, and bundle rewards; verify service-area coverage before recommending.
8. For healthcare/pharmacy, search both commercial pharmacy loyalty, rewards, and prescription savings programs, and official government healthcare, coverage, or prescription benefit programs; clearly label commercial versus government/public programs.

## Output Rules

- Each candidate must include program name, provider, category, source URL when available, and a short evidence summary.
- Prefer local or service-area-specific pages over generic national pages.
- Prefer named programs over vague categories.
- Do not invent program names, URLs, rewards, fees, or eligibility.
